# Reporting Seizures and False Alarms

Once Data Sharing has been set-up correctly (see [Data Sharing Set-up Instructions](/pages-user/data-sharing/data-sharing-setup.html)), the app will check the database periodically to see if there are events uploaded that have not been marked as genuine seizures or false alarms.

If there are un-confirmed events, an extra notification will be shown on the top of the phone screen (the OpenSeizureDetector icon with a question mark '?' in front of it).

Selecting the notification will take you to the Events List screen. All events are listed on the screen with unconfirmed events highlighted in pink (Confirmed real seizures are shown in red).

To mark an event, select it from the list. This will open an Edit Event screen. Select the event type (Seizure, other medical issue, false alarm) and a sub-type (which depends on the event type). If there is not a suitable sub-type shown to describe what was happening to cause the event, select 'Other' and add some notes into the notes box at the bottom of the screen.

Press 'Save' to save the event to the database.
